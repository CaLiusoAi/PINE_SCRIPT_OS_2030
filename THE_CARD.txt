THE 15 HARD LAWS (each one is a compiler kill or repaint trap)

  L3  bool is two-state. NEVER `bool x = na`. NEVER na()/nz()/fixnan() on

      ONLY with a [1]-offset source (the confirmed-HTF pattern).
  L7  All state mutation in ONE block gated by barstate.isconfirmed.
  L8  Objects (line/box/label/table): create under barstate.isfirst,
      update under barstate.islast. Respect max_*_count.
  L9  input.int/input.float: minval, maxval, step on every one.
  L10 plot/plotshape/bgcolor/alertcondition: const string titles; these

  L13 Division: 5/2 == 2.5 in v6. int(a/b) when truncation intended.
      Guard every divisor: na/zero check before dividing.
  L14 timeframe.period includes multiplier: compare "1D" never "D".
      UDT history: (obj[1]).field never obj.field[1].

── PART B · IMAGE LAW (images are ground truth) ──

IRTS FORCING CARD v1.1 — PREFIX THIS TO EVERY IMAGE→CODE REQUEST
Works on any AI instance. Removes the shortcut; does not rely on capability.
Source: PineAgent_v6_IRTS.md (canon) + Phase 1.5 human gate (session-learned)
═══════════════════════════════════════════════════════════════════════════

Images are ground truth. Code is a hypothesis to falsify against them.
Indicator names, pattern labels, and my annotations' WORDS are noise —
the PIXELS and PRICES are the spec. Synthesis is topologically downstream
of observation. You cannot skip ahead.

MANDATORY SEQUENCE — each phase is a separate message turn:

PHASE 1 — EXTRACT (no code, no interpretation)
  Numbered table: OBS-01..OBS-nn. One measurable fact per line:
  exact prices, body/wick sizes and ratios, candle colors, bar positions,
  line/marker positions, hover-data OHLC, symbol, timeframe.
  Tag each line [READ] (directly legible) or [EST] (estimated from pixels).
  STOP. Output nothing else.

PHASE 1.5 — HUMAN GATE (the anti-butchering lock)
  The human confirms or corrects each OBS line.
  Only the human-corrected table is citable downstream.
  If the human says an OBS is wrong, it is wrong. No defense.
  ⛔ No model may proceed past this line without explicit human approval.

PHASE 2 — FALSIFY (no code)
  Every candidate/existing condition → PASS / FAIL / AMBIGUOUS
  against the APPROVED table only. One-line reason each.

PHASE 3 — MAP (no code)
  Pseudocode lines, each citing OBS numbers from the approved table.
  No citation → UNSUPPORTED → deleted. AMBIGUOUS items become
  questions to the human, not assumptions.

PHASE 4 — SYNTHESIZE
  Pine v6 code. EVERY condition carries a trailing comment:
  // MAP-xx ← OBS-yy
  Code with any uncited condition is invalid and must be rejected.
  Run pine_preflight.py --irts on it: uncited condition = FAIL.

NUMERAL RULE (anti observation-theater): every [READ] OBS row must
  contain at least one literal number taken from the image (price, time,
  ratio, bar count). An extraction with zero numerals is VOID on sight.
GATE-FINALITY RULE: Phase 1.5 must be the LAST line of its message. Any
  message containing Phase 1.5 plus later phases is VOID — the gate is a
  wall, not a decoration around already-written code.
PREFLIGHT-AUTHENTICITY RULE: preflight reports are pasted verbatim from
  execution; gate IDs must exist in the registry (G01–G15). A report with
  unknown gate IDs or no FAIL/PASS exit line is a fabrication (ERR-004).

REJECTION RULE (the human enforces this):
  Model outputs code before Phase 1.5 approval → reject, restart Phase 1.
  Model lectures about anything not in the OBS table → reject, restart.
  Model cites an OBS that doesn't exist → reject, restart.
